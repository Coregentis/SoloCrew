# SOLOCREW v0.5 Non-Goals v0.1

## Purpose

This audit freezes what `v0.5-portfolio-secretary-beta` does not include.

It exists to keep `v0.5` meaningful as the first bounded Secretary and portfolio beta line instead of letting it collapse into `v1.0` breadth.

## Non-Goals

| Non-Goal | Why It Is Out Of Scope For `v0.5` | Current Boundary |
| --- | --- | --- |
| Full portfolio operating platform | `v0.5` is a bounded beta line, not the finished platform line. | Deferred beyond `v0.5`. |
| Enterprise team collaboration | SoloCrew remains a solo-operator repo at this stage. | Explicitly absent. |
| Deep org permissions model | `v0.5` does not become an enterprise governance or RBAC line. | Explicitly absent. |
| Multi-tenant runtime | Runtime tenancy remains outside SoloCrew product scope here. | Explicitly absent. |
| Provider/channel execution platform | `v0.5` may coordinate, but it does not become an execution platform. | Explicitly absent. |
| Broad KPI cockpit | `v0.5` is not a broad analytics or executive cockpit line. | Explicitly absent. |
| Autonomous operator-grade company runtime | `v0.5` does not claim full autonomous company operation. | Explicitly absent. |
| MPLP promotion | Downstream product scope does not promote protocol law. | Must remain outside SoloCrew. |
| Runtime authority ownership | `Cognitive_OS` remains the mother-runtime provider of runtime-private truth. | Must remain outside SoloCrew. |
| Full `v1.0` behavior | `v0.5` is a bounded beta, not the finished SoloCrew platform. | Deferred to future `v1.0`. |

## Non-Goal Rule

If a proposed `v0.5` implementation mainly serves:

- enterprise administration
- full orchestration engine behavior
- direct execution platform breadth
- autonomous operating-company behavior
- runtime or protocol promotion

then it is not a valid `v0.5` change by default.
