# SoloCrew V1.1 Intake-to-Packet Test Plan v0.1

`SOLOCREW-V1.1-INTAKE-TO-PACKET-TEST-PLAN-v0.1`

## Purpose

This plan defines the required future tests for the V1.1 intake-to-packet
implementation wave.

## Required Future Tests

- founder request maps to packet candidate
- projection-safe state exposure maps without approval semantics
- terminal maps without execution-complete semantics
- evidence summary maps without proof/certification semantics
- stale and insufficient evidence stay distinct
- non-executing recommendation remains non-executing
- forbidden execution labels are rejected
- raw runtime-like keys are rejected
- project mismatch between request and projection summary is rejected
- packet candidate remains review/staging only
- scenario fixture produces expected packet posture
- no provider/channel or queue semantics appear
