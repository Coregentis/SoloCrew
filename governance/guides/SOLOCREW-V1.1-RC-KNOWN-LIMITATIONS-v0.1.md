# SoloCrew V1.1 RC Known Limitations v0.1

`doc_id: SOLOCREW-V1.1-RC-KNOWN-LIMITATIONS-v0.1`

## A. Purpose

This guide documents the known limitations of SoloCrew V1.1 after RC release.

## B. Limitations Table

| Limitation | Why it exists | User impact | Future direction |
|---|---|---|---|
| no provider/channel execution | V1.1 RC is intentionally non-executing | users can review but not send externally | future bounded product hardening or later product lines |
| no approve/reject/dispatch/execute | control semantics remain out of scope | users see posture, not command buttons | future product decisions only if separately authorized |
| no founder queue | queue semantics were intentionally excluded | users do not get queued founder workflow behavior | possible future product line exploration |
| no live external workflow | RC is scoped to internal review visibility | no end-to-end external action loop | later product planning only |
| no autonomous company operation | product remains far below autonomy claim | users should not expect self-operating behavior | out of scope for V1.1 RC |
| no protocol certification | release remains below certification semantics | no certification claim is implied | protocol matters remain upstream-only |
| no Cognitive_OS runtime-private consumption | SoloCrew remains a downstream product consumer | users see product-safe projections only | future improvements remain downstream-consumption only |
| no MPLP schema/protocol change | this RC does not alter upstream law | users should not expect protocol evolution from this release | future backflow only as evidence/candidate notes |
| no persistence/queue beyond existing bounded state | V1.1 loop is review/staging oriented | no broader workflow durability beyond current bounded state | future product hardening if needed |
| RC is not GA | this is a prerelease line | users should treat this as bounded RC behavior | later release decisions if warranted |

## C. Non-bug Clarification

These limitations are intentional RC boundaries, not defects.
