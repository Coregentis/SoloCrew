# SOLOCREW-SINGLE-CELL-INIT-BASELINE-v0.1

## Purpose

This document freezes the first-user initialization path for SoloCrew after `v0.1` closure.

The initialization target is one usable `Cell`, not a portfolio system.

## Initialization Baseline

The first-user initialization path is:

1. one operator
2. one `Cell`
3. one `CEO Orchestrator`
4. one native multi-agent crew
5. one current objective
6. one `Cell Console`

This baseline should remain true even if later versions add portfolio or Secretary layers.

## Initialization Objects

Single-cell initialization should establish, at minimum:

- one `Cell Charter`
- one `Delivery Contract`
- one `Cell Policy Profile`
- one `CEO Orchestrator Contract`
- one `Crew Blueprint`
- one `Objective Portfolio` with one current objective
- one narrow `Execution Ledger` posture
- one narrow `Memory & Evidence Anchor` posture

Optional mounts may remain absent:

- `Business Pack Mount`
- `Metrics Pack Mount`

## Day-One Product Shape

The first visible shape after initialization should be a `Cell Console`.

The `Cell Console` should be the place where the operator sees:

- the crew
- the current objective
- the active and blocked work
- the bounded review/correction surface
- the continuity/reload truth

Initialization must not require a broad management dashboard before this `Cell Console` exists.

## Day-One Structural Commitments

Day one commits to:

- one working operating unit
- one management-to-cell path, even if minimal
- one current objective at a time
- honest continuity over stored truth
- bounded crew compilation and execution posture

## Intentional Deferrals

This baseline intentionally defers:

- multi-cell initialization
- portfolio-first setup
- full Secretary implementation
- broad KPI cockpit setup
- provider or channel integration
- budget runtime
- business pack execution logic
- metrics pack execution logic

## Initialization Rule

If a proposed future setup flow cannot create a valid single `Cell` cleanly, it should not be treated as the mainline initialization flow.
