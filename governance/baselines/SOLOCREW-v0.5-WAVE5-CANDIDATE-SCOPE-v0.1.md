# SOLOCREW v0.5 Wave 5 Candidate Scope v0.1

## Candidate Wave Name

`v0.5 Wave 5 - rationale and evidence visibility hardening`

## Purpose

This document defines the smallest lawful next SoloCrew wave after reopening
approval.

The wave remains intentionally narrow:

- downstream
- product-projected
- non-executing
- explanation-first

## Why This Is The Next Safe Wave

Current upstream now supports:

- sealed bounded Phase 4 mother-runtime truth
- bounded `Confirm` / `Trace` export posture
- lawful `Context` / `Plan` omission

That makes a rationale/evidence visibility wave safe because it can improve
downstream interpretation without pretending:

- upstream workflow-state adoption
- upstream canonical `Context` / `Plan` completeness
- direct-control authority

## Allowed Scope

This candidate wave may introduce only bounded downstream product work such as:

- rationale visibility on existing Secretary/portfolio surfaces
- evidence visibility tied to current bounded upstream `Confirm` / `Trace`
  posture
- clearer explanation of why a packet currently reads as:
  - `draft`
  - `staged`
  - `ready_for_cell_review`
  - `returned_for_revision`
- bounded review narration / interpretation surfaces
- bounded portfolio-level summary polish that clarifies status, rationale, and
  evidence posture

## Explicitly Forbidden

This candidate wave must not introduce:

- direct approve controls
- direct reject controls
- direct dispatch controls
- direct execute/provider/channel controls
- workflow-engine semantics
- runtime authority ownership
- protocol authority ownership
- local imitation of a missing upstream workflow-state family
- local claims that canonical MPLP `Context` / `Plan` export already exist

## Upstream Assumptions

This candidate wave may assume:

- bounded `Confirm` / `Trace` export posture exists upstream
- those upstream surfaces remain bounded and non-promoting
- `Context` / `Plan` omission remains lawful
- packet states remain downstream-only product semantics

This candidate wave must not assume:

- upstream packet-state workflow truth exists
- upstream full protocol completeness exists
- MPLP has promoted any SoloCrew-facing semantics into protocol law

## Exit Condition

The wave is acceptable only if it remains:

- explanatory rather than controlling
- evidence-facing rather than execution-facing
- product-projected only
- clearly below direct-control semantics

## Net Scope Rule

The correct reading of Wave 5 candidate scope is:

- improve clarity
- improve interpretation
- improve evidence/rationale visibility
- do not widen authority
