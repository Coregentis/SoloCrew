# SOLOCREW-VERSION-ROADMAP-v0.1

## Purpose

This document freezes the post-`v0.1` version sequence for SoloCrew so structural work proceeds in coherent waves instead of feature drift.

## Version Sequence

### `v0.2-structural-constitution`

Goal:

- freeze the structural constitution of SoloCrew before implementation breadth expands

Mandatory baseline:

- `SOLOCREW-STRUCTURAL-CONSTITUTION-v0.1`
- `SOLOCREW-CELL-CONSTITUTION-v0.1`
- `SOLOCREW-CREW-COMPILER-CONTRACT-v0.1`
- `SOLOCREW-MANAGEMENT-INTERFACE-BOUNDARY-v0.1`
- `SOLOCREW-SINGLE-CELL-INIT-BASELINE-v0.1`
- `SOLOCREW-ARCHITECTURE-PLANES-v0.1`
- `SOLOCREW-UPSTREAM-BLOCKER-LEDGER-v0.1`

Explicitly deferred:

- runtime expansion
- provider integration
- channel integration
- budget implementation
- broad UI implementation
- business-pack execution logic

### `v0.3-single-cell-usable`

Goal:

- make one `Cell` genuinely usable as a first product form over the already sealed `v0.1` truth

Mandatory baseline:

- single-cell initialization path
- one working `Cell Console`
- one active objective surface
- work-item tracking
- bounded correction/review surface
- honest continuity/reload experience

Explicitly deferred:

- multi-cell portfolio management
- full Secretary surface
- broad KPI cockpit
- external channels
- budget runtime

### `v0.5-portfolio-secretary-beta`

Goal:

- add a bounded management-plane expansion over multiple `Cell`s

Mandatory baseline:

- `Management Directive` surface
- `Cell Summary Card` surface
- `Escalation / Approval Request` flow
- `Delivery Return` flow
- bounded portfolio and Secretary beta shape

Explicitly deferred:

- enterprise analytics cockpit
- full autonomous portfolio optimization
- generalized pack marketplace
- protocol promotion of product objects without separate candidate handling

### `v1.0-solo-operator-platform`

Goal:

- deliver SoloCrew as a coherent solo-operator business platform with stable structural layers

Mandatory baseline:

- stable single-cell operating core
- bounded portfolio/Secretary layer
- lawful business-pack and metrics-pack mount model
- coherent management, organization, execution, and memory/evidence planes
- explicit upstream boundary discipline preserved

Explicitly deferred:

- anything that would collapse product, runtime, and protocol authority into one layer
- speculative expansion that has not survived blocker-ledger review

## Roadmap Rule

Each version must preserve:

- `MPLP Protocol -> Cognitive_OS -> SoloCrew`
- one coherent wave at a time
- no silent promotion of product structure into runtime or protocol authority
