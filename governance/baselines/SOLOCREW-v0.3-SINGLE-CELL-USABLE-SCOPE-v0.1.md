# SOLOCREW v0.3 Single-Cell Usable Scope v0.1

## Purpose

This document freezes the product boundary for `v0.3-single-cell-usable` before any usable-product implementation work begins.

`v0.3` is the first real usable single-cell product line.
It is not a portfolio release, not a Secretary release, and not a broad cockpit release.

## v0.3 Goal

The goal of `v0.3-single-cell-usable` is:

- one operator
- entering one `Cell`
- through one actual product entry surface
- into one usable `Cell Console`
- with bounded work, correction/review, and continuity truth

The usable threshold is practical, not architectural alone.
The product must become operable by one person inside one real single-cell flow.

## Must Be Included

`v0.3-single-cell-usable` must include:

- one actual single-cell entry surface built over the sealed shell entry adapter
- one actual single-cell console surface built over the sealed console/view-model/shell scaffolds
- one visible current objective surface
- one visible work-item or workstream tracking surface
- one visible correction/review surface using the already sealed bounded `v0.1` truths
- one visible continuity/reload surface that stays honest about what is and is not persisted
- explicit visibility of deferred surfaces so the product does not pretend missing capability exists

## Structural Inputs To Reuse

`v0.3` must reuse the already sealed `v0.2` structural ladder:

- structural object layer
- management contract layer
- single-cell structural assembly scaffold
- single-cell console state scaffold
- single-cell view-model scaffold
- single-cell shell composition scaffold
- single-cell shell entry adapter scaffold

## Product Rule

`v0.3` should turn the current single-cell scaffolds into one usable operator-facing flow.

It should not reopen:

- structural constitution debates
- runtime authority order
- product vs runtime vs protocol boundaries

## Remains Deferred

`v0.3-single-cell-usable` still defers:

- multi-cell portfolio behavior
- Secretary behavior
- provider execution
- channel integrations
- broad KPI cockpit behavior
- broad business-pack execution breadth
- runtime-complete product state

## Scope Judgment Rule

If a proposed `v0.3` implementation does not make one single `Cell` honestly usable by one operator, it does not satisfy this scope.

If a proposed `v0.3` implementation drifts into portfolio, Secretary, or cockpit breadth before that threshold is met, it violates this scope.
