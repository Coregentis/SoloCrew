# SOLOCREW Single-Cell Usable Contract v0.1

## Purpose

This contract freezes the minimum usable-product contract for `v0.3-single-cell-usable`.

It defines the minimum truth SoloCrew must support before claiming a first real usable single-cell product line.

## Minimum Usable Contract

### 1. One Operator

The product must support one operator as the primary user.

Minimum contract:

- one operator enters the product
- one operator sees one active `Cell`
- one operator can understand what action is needed next

### 2. One Cell

The product must support one structurally valid `Cell`.

Minimum contract:

- one `Cell Charter`
- one `Delivery Contract`
- one `Cell Policy Profile`
- one `CEO Orchestrator Contract`
- one `Crew Blueprint`
- one `Objective Portfolio`
- one `Execution Ledger` posture
- one `Memory & Evidence Anchor` posture

### 3. One Crew

The product must support one bounded crew view for the active `Cell`.

Minimum contract:

- visible crew framing
- visible required roles
- visible bounded compile posture

This does not require live provider orchestration.

### 4. One Current Objective

The product must support one visible current objective for the active `Cell`.

Minimum contract:

- current objective headline
- current objective identity
- clear current focus

### 5. Work-Item Tracking

The product must support bounded work tracking.

Minimum contract:

- visible active work truth
- visible blocked work truth
- visible counts or bounded work-item/workstream state

This does not require a full event timeline.

### 6. Correction / Review

The product must support bounded correction and review behavior already justified by sealed `v0.1` truth.

Minimum contract:

- an operator can issue a bounded correction
- the product can show bounded review posture
- continuity and writeback implications remain explicit and honest

### 7. Continuity / Reload Truth

The product must support honest continuity and reload behavior.

Minimum contract:

- the operator can return to the same `Cell`
- persisted truth remains visible where it is actually stored
- missing truth remains explicitly absent where it is not stored

## Explicit Non-Claims

`v0.3-single-cell-usable` does not claim:

- multi-cell portfolio behavior
- Secretary behavior
- provider execution
- channel integrations
- broad KPI cockpit behavior
- broad business-pack execution breadth
- runtime-complete product state
- silent elevation of product law into runtime or protocol authority

## Contract Rule

If any part of the product experience requires pretending missing truth exists, the single-cell usable contract is not satisfied.
