## 1. 产品定义

### 1.1 一句话定义

**SoloCrew 是一个给独立开发者、一人公司、超级个体使用的 24/7 Agent 员工班底，让用户像拥有一支会接力、会成长、越来越像老员工的数字团队。**

### 1.2 产品不是

SoloCrew 不是：

* 一个单 Agent 聊天工具
* 一个自动化 flow 拼装器
* 一个 IDE Copilot 壳
* 一个协议展示层
* 一个抽象的 Agent OS 概念站

### 1.3 产品主语

SoloCrew 的产品主语是：

> **“一个人拥有的一支持续工作的 Agent 团队”**

用户购买的是：

* 团队感
* 持续感
* 接力感
* 老员工感
* 业务推进感

不是购买“更复杂的架构”或“更多 AI 概念”。

## 2. 目标用户定义

### 2.1 核心 ICP

#### ICP-A：独立开发者 / Solo Founder

同时承担开发、增长、内容、运营、客户跟进，希望 AI 不只是辅助，而是成为持续执行的数字员工。

#### ICP-B：AI Power User

已经重度使用 ChatGPT、Claude、Cursor、OpenClaw、Hermes、自动化工具，但强烈感受到：

* 不连续
* 不共享脑子
* 多 Agent 反而更乱
* 长期成本失控
* 不像真正团队

#### ICP-C：一人公司 / 小型工作室操盘手

已有明确业务目标与收入诉求，希望 AI 成为实际产能来源，而不是“看起来很聪明”。

### 2.2 非首批目标用户

v0.1 暂不优先服务：

* 纯娱乐型用户
* 只想尝鲜 AI 的轻度用户
* 没有明确业务目标的“躺赚幻想型”用户
* 大型企业复杂治理场景

---

## 3. 用户核心目标

### 3.1 用户真正想买的结果

用户不是想买一个“AI 工具”，而是想买下面这些结果：

1. **持续工作**
   AI 团队能 7×24 小时持续推进目标，不是每次都重新开始。

2. **团队分工**
   不是一个万能 bot，而是一组有职责分工的数字员工。

3. **跨天接力**
   昨天布置的事，今天还能接着做，不失忆、不重启。

4. **老员工感**
   越用越懂用户业务、目标、偏好、关键词、习惯。

5. **可控而不费人**
   用户能插手、能纠偏、能定目标，但不需要亲自执行所有细节。

6. **可见回报**
   在开发、增长、内容、运营上形成真实产能与结果。

## 4. 产品核心价值主张

### 4.1 核心价值

SoloCrew 的核心价值不是“AI 很强”，而是：

> **让一个人拥有一支真实可持续工作的数字员工班底。**

### 4.2 对比市场现有工具的用户感知差异

#### 对比普通聊天 AI

不是“回答更聪明”，而是“可以持续推进工作”。

#### 对比普通自动化工具

不是“拼流程”，而是“像员工一样协同工作”。

#### 对比单 Agent 助手

不是“一个助手更强”，而是“形成班底和接力”。

#### 对比现有多 Agent 工具

不是“多几个 Agent”，而是：

* 共用一个项目脑子
* 不容易越跑越乱
* 会越来越像老员工

---

## 5. v0.1 要解决的核心问题

### 5.1 当前用户痛点

用户现在最痛的，不是“AI 不够聪明”，而是：

1. **不连续**
   每次使用都像重新开始。

2. **不成团队**
   多个 Agent 只是多个聊天框，没有真正协作。

3. **不共享脑子**
   内容、增长、开发之间无法共享事实与目标。

4. **不够像员工**
   用户每次都要重新解释风格、优先级、业务背景。

5. **不够可控**
   要么需要人盯死，要么放开后就容易跑偏。

6. **成本不可持续**
   长上下文、多 Agent、重复读取导致 token 消耗过高。

### 5.2 v0.1 的解决承诺

SoloCrew v0.1 不承诺“全自动赚钱”，但必须承诺：

* 用户第一次能明显感到“我拥有一支 Agent 团队”
* 这支团队围绕目标持续工作，而不是一次性回答
* 用户纠正之后，团队会开始表现出“老员工化”趋势

## 6. SoloCrew v0.1 功能抽象

### 6.1 核心功能域一：Crew 班底管理

用户能够拥有并管理自己的数字员工班底。

**功能点**

* 创建 Crew
* 查看 Crew 总览
* 预设员工角色
* 激活 / 暂停员工
* 查看员工当前状态
* 查看员工最近动作与产出

**首批预设角色**

* Builder
* Growth
* Content
* Ops

---

### 6.2 核心功能域二：目标驱动工作

用户不是给 prompt，而是给目标。

**功能点**

* 创建 Objective
* 将目标分配给不同员工
* 员工围绕目标推进工作
* 用户查看目标推进状态
* 用户可手动纠偏

---

### 6.3 核心功能域三：持续接力

用户离开后，团队仍然围绕目标继续推进。

**功能点**

* 任务状态保留
* 员工之间接力
* 次日继续推进
* 中断恢复
* 汇报用户需要决策的事项

---

### 6.4 核心功能域四：老员工感记忆

用户与团队交互越多，团队越贴近用户偏好。

**功能点**

* 记住用户偏好
* 记住关键词 / hashtag / 业务词汇
* 记住角色习惯
* 基于纠正进行调整
* 下次更贴近用户风格与判断

---

### 6.5 核心功能域五：至少一个外部沟通入口

SoloCrew 不能只存在于一个网页里。

**功能点**

* Web Cockpit
* 至少一个消息入口（建议 Telegram 先行）
* 统一消息归档与状态映射
* 通知用户关键事件

---

### 6.6 核心功能域六：基础结果与成本可见

用户要感知到这支团队不是“空转”。

**功能点**

* 员工最近做了什么
* 哪个目标推进了多少
* 今日 / 本周汇报
* 基础 token / 预算可见
* 基础异常提示

## 7. Wow Moment 定义

### 7.1 SoloCrew 的产品 wow 不是什么

不是：

* 模型一次答得很惊艳
* 某个单点自动化 flow 跑通
* 某个 Agent 看起来很聪明

这些都不是 SoloCrew 的核心 wow。

### 7.2 主 Wow Moment

**用户第一次打开 SoloCrew，就看到一支围绕自己目标持续工作的 Agent 团队，而不是一个聊天机器人。**

这意味着：

* 有明确员工角色
* 有正在推进的工作
* 有彼此之间的接力关系
* 有统一目标
* 有状态变化
* 有“这真的是我的班底”的感受

### 7.3 次级 Wow Moment

**第二天回来，团队还记得昨天，并且已经继续推进工作。**

用户感知：

* 没失忆
* 没重置
* 没让我重新解释
* 真的像在持续上班

### 7.4 第三级 Wow Moment

**用户纠正过一次之后，团队下次更像“带出来的老员工”。**

用户感知：

* 它吸收了我的风格
* 它记住了我的优先级
* 它不是每次犯同样的错误

## 8. v0.1 成功标准

### 8.1 产品级成功标准

v0.1 成功，不是功能越多越好，而是以下体验是否成立：

1. 用户第一次进入时，是否感知到“团队感”
2. 用户布置目标后，是否感知到“持续推进”
3. 用户第二次回来时，是否感知到“接力与记忆”
4. 用户纠正后，是否感知到“老员工萌芽感”

### 8.2 业务级成功标准

v0.1 首批用户应至少能在以下之一看到价值：

* 开发任务推进更连续
* 内容生产更连续
* 增长动作更持续
* 决策和跟进压力降低
* 对 AI 团队的信任增强

### 8.3 失败信号

若出现以下情况，则说明 v0.1 没打中：

* 用户觉得只是“多几个聊天框”
* 团队不能跨天接力
* 用户每次都要重新解释背景
* 老员工感没有出现
* 用户看不到任何业务推进或工作状态
* 成本结构明显不可持续

## 9. v0.1 产品边界

### 9.1 必须做

* Crew 创建与总览
* 4 个基础员工角色
* Objective 创建与分配
* 基本任务推进与接力
* Web Cockpit
* 至少 1 个消息入口
* 基础状态持久化
* 基础共享记忆
* 用户偏好捕获
* 基础预算 / token 可见
* 基础 review / 汇报

### 9.2 应该做

* 简化 KPI / OKR
* 每日汇报
* 基本纠偏机制
* 至少 1 个外部 Agent 接入骨架
* 基础异常与失败提示

### 9.3 暂不做

* 完整跨组织平台
* 完整 Team / Enterprise 能力
* 完整自动扩缩
* 完整 Agent marketplace
* 完整深度学习系统
* 完整行业化模板矩阵
* “全自动赚钱”式结果承诺

## 10. 首批体验流

### Flow-01：创建自己的 Crew

用户进入产品，看到默认 4 名员工角色并完成初始配置。

### Flow-02：下达当前目标

用户输入一个明确目标，例如：

* 本周把 landing page + 内容 + 线索跟进跑起来
* 推进某功能开发并发布更新
* 做某个主题的内容分发与客户触达

### Flow-03：团队开始工作

不同员工开始围绕目标分工：

* Builder 拆任务
* Content 准备内容
* Growth 找线索
* Ops 汇总与提醒

### Flow-04：用户纠偏

用户对其中一位员工进行偏好纠正、优先级调整、关键词补充。

### Flow-05：系统体现老员工感

下一轮相关动作中，该员工和团队整体开始更贴近用户偏好。
