## 0. 先给硬裁决

按你已经冻结的路线，**这次不能把 SoloCrew 全部写进 `Coregentis/Cognitive_OS` 仓库**。

因为当前公开仓库 README 已经把 `Cognitive_OS` 明确定义为 **neutral mother-runtime foundation repository**，并且明确写了：

* 这里承载的是 `governance/ imports/ schemas/ registry/ bindings/ runtime/ tests/` 这些基础层；
* **product-specific or application-specific implementation** 不属于这里；
* **product projections** 和下游应用应放在独立仓库；
* authority order 明确是 **`MPLP Protocol Constitution -> Coregentis Cognitive OS -> Product Projections -> TracePilot`**。([GitHub][1])

所以这份蓝图的第一原则是：

> **Schema / Runtime 的“通用、产品无关、可复用”部分进 `Cognitive_OS`；
> Projection / App 的“SoloCrew 专属”部分必须拆到下游独立仓库。** ([GitHub][1])

# 1. 仓库拆分总图

## 1.1 现有仓库真相

`Coregentis/Cognitive_OS` 当前顶层已有这些目录：
`OS_core_file/`, `bindings/`, `governance/`, `imports/`, `registry/`, `runtime/`, `schemas/coregentis/v0/`, `tests/`，README 也把这些列为 v0.1 foundation baseline 的 home。([GitHub][1])

## 1.2 正确的 SoloCrew 仓库落位

按这条边界，建议拆成三层：

### A. 上游约束源

* `Coregentis/MPLP-Protocol`
* 不改产品实现，只引用其协议语义与 schemas。([GitHub][1])

### B. 母运行时基础仓库

* `Coregentis/Cognitive_OS`
* 只放 **通用 Schema / 通用 Runtime / 通用 Tests / Governance 记录**

### C. 下游产品仓库

建议新增两个独立仓库：

* `Coregentis/SoloCrew_Projection`
* `Coregentis/SoloCrew_App`

如果你想更极致收敛，也可以先只建一个：

* `Coregentis/SoloCrew`

但按你自己冻结的 **MPLP → Cognitive → Projection → App** 路线，**拆成 Projection / App 两仓更干净**。

# 4. 仓库级禁止混放规则

这个要写死，否则后面一定会乱。

## 4.1 不允许放进 `Coregentis/Cognitive_OS` 的内容

* SoloCrew 专属 UI 组件
* SoloCrew 页面路由
* App DTO
* 产品页状态管理
* 产品 marketing 文案
* Telegram 交互 UI 逻辑
* 任何“只为 SoloCrew 页面存在”的对象

这条边界和当前仓库 README 的 “not as an application or product surface” 完全一致。([GitHub][1])

## 4.2 可以放进 `Coregentis/Cognitive_OS` 的内容

* 通用 workforce schema
* 通用 worker lifecycle runtime
* 通用 graph/state/action runtime
* 通用 adapter/channel contract
* 通用 tests / governance / audit artifacts

# 6. 一句话收敛

**按仓库现实，SoloCrew v0.1 的正确拆法是：把“可复用、可约束、可承载产品体验的通用能力”放进 `Cognitive_OS`，把“SoloCrew 专属的对象投影、页面、交互和产品壳”下沉到独立 Projection / App 仓库。**

这既符合你冻结的
**MPLP → Cognitive → Projection → App**
路线，也符合 `Cognitive_OS` 当前 README 已公开写死的仓库边界。([GitHub][1])
